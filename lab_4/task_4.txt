# Лабораторная работа

## Проектирование системы с партиционированием и

## шардированием

Спроектировать архитектуру базы данных для системы учёта доходов и расходов с
использованием: партиционирования, шардирования, репликации.

## Задание 1: Проектирование базы данных

Спроектировать структуру БД системы “Кошелёк расходов/доходов”.

**Необходимо определить:**

- таблицы (пользователь, счёт, категории, транзакции, агрегаты)
- поля таблиц
- связи между сущностями

**Результат:**

- описание таблиц или ER-диаграмма

## Задание 2: Партиционирование

Выбрать стратегию партиционирования для таблицы с наибольшим объёмом данных.

**Требуется:**

- определить тип
- выбрать критерий
- описать структуру партиций

**Обосновать выбор и указать, какие запросы ускоряются.**

## Задание 3: Шардирование

Спроектировать распределение данных между несколькими серверами.

**Требуется:**

- выбрать способ
- определить ключ
- описать алгоритм распределения данных

**Указать:**

- количество шардов
- как определяется нужный шард
- где хранятся данные пользователя


## Задание 4: Репликация

Добавить репликацию для обеспечения отказоустойчивости.

**Необходимо:**

- описать структуру
- указать:
    o куда идут записи
    o откуда читаются данные

**Кратко описать поведение при сбоях.**

## Задание 5: Архитектура системы

Описать общую архитектуру системы.

## Задание 6: Анализ

Кратко ответить:

1. Почему выбран данный способ шардирования?
2. Какие преимущества даёт партиционирование?
3. Возможные проблемы системы
4. Способы масштабирования (добавление шардов, балансировка)
