# STRIS Labs

Репозиторий с лабораторными работами по предмету **Modern Technologies for the Development of Information Systems**.

Короткое название для GitHub: **`stris-labs`**.

Расшифровка: **СТРИС** — современные технологии разработки информационных систем. В английском варианте можно использовать **STRIS** как компактное имя репозитория и проекта.

## О предмете

Предмет посвящен современным подходам к проектированию и разработке информационных систем. В лабораторных работах рассматриваются практические темы, которые часто встречаются в backend-разработке, распределенных системах и инфраструктуре:

- проектирование REST API и описание контрактов через OpenAPI;
- HTTP-сервисы, reverse proxy, балансировка нагрузки и кэширование;
- работа с базами данных, индексами, транзакциями и уровнями изоляции;
- распределенное хранение данных, репликация, шардирование и партиционирование;
- событийная архитектура и брокеры сообщений на примере Kafka;
- архитектурные и инфраструктурные паттерны web-систем.

Главная цель курса — показать, как отдельные технологии объединяются в надежные, масштабируемые и сопровождаемые информационные системы.

## Лабораторные работы

| Лабораторная | Тема | Что выполняется |
| --- | --- | --- |
| [`lab_0`](./lab_0) | Проектирование REST API | Создается спецификация OpenAPI для API социальной сети: пользователи, друзья, посты, форматы ответов и ошибок. |
| [`lab_1`](./lab_1) | HTTP-сервис, reverse proxy, балансировка, кэш | Поднимаются два экземпляра сервиса, Nginx распределяет запросы между ними, Redis используется для кэширования данных. |
| [`lab_2`](./lab_2) | Индексы, транзакции, уровни изоляции, Cache Aside | На PostgreSQL демонстрируются индексы, rollback транзакций и уровни изоляции; Redis используется как внешний кэш. |
| [`lab_3`](./lab_3) | Распределенное хранение данных | Реализуется простая master-replica репликация в памяти, проверяется поведение системы при остановке и восстановлении реплики. |
| [`lab_4`](./lab_4) | Партиционирование, шардирование, репликация | Проектируется база данных кошелька доходов/расходов с партициями по датам, шардированием по пользователю и чтением из реплик. |
| [`lab_5`](./lab_5) | Kafka и событийная обработка | Настраивается Kafka topic с партициями, producer отправляет JSON-сообщения, consumer group распределяет обработку между consumer-ами. |
| [`lab_6`](./lab_6) | Паттерны проектирования web-систем | Реализуется marketplace-сценарий обработки платежных заказов с CQRS, Pub/Sub, Service Discovery, Circuit Breaker, Backpressure, ETag, очередями и SSE. |

## Технологии

- Docker и Docker Compose
- Python

- Nginx
- PostgreSQL
- Redis
- Kafka и Zookeeper
- OpenAPI/Swagger

## Как запускать

Каждая лабораторная работа находится в отдельной папке и содержит собственный `README.md` с командами запуска и проверки.

Обычно запуск выполняется так:

```bash
cd lab_N
docker compose up --build
```

Для остановки контейнеров:

```bash
docker compose down
```
