# Лабораторная работа 3

## Тема: распределенное хранение данных (репликация)

## Задание 1. Простая репликация

1. Создать систему из 3 узлов:
   - Master (Node A)
   - Replica-1 (Node B)
   - Replica-2 (Node C)
2. Master принимает записи и рассылает их на реплики.
3. Все данные хранятся в памяти (`key-value` словарь), без БД.

### Endpoint'ы

Master:
- `POST /data?mode=sync` - запись и репликация на Replica
- `GET /data/{key}` - получение значения по ключу

Replica:
- `POST /replica/data` - прием данных от Master
- `GET /data/{key}` - чтение значения по ключу

## Задание 2. Имитация сбоя

1. Отключить одну Replica.
2. Сделать несколько записей через Master (`POST /data`).
3. Проверить:
   - какие данные есть на Master и активной Replica;
   - что будет после восстановления отключенной Replica;
   - есть ли расхождения между узлами.
