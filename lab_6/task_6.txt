# Лабораторная работа

# Тема: Паттерны проектирования в Web Design

Спроектировать веб-систему с использованием современных архитектурных и
инфраструктурных паттернов.

# Задание

Спроектировать систему (например: финансы, маркетплейс, соцсеть), применив
указанные паттерны.

# Требования

## 1. Архитектура

- использовать **трёхзвенную архитектуру**

Client → Backend → Database

- определить тип клиента (толстый/тонкий)

## 2. Взаимодействие сервисов

Использовать:

- Pub/Sub
- Service Discovery
- Heartbeat

## 3. Работа с данными

- применить CQRS
- описать обработку данных (MapReduce)

## 4. Надёжность

Реализовать:

- Retries + Backoff
- Idempotency
- Circuit Breaker
- Backpressure

Обработка ошибок:

- Fallback
- Graceful Degradation


## 5. Кэширование

- версионирование кэша
- тегирование кэша

## 6. Асинхронность

- отложенные задачи
- очередь сообщений

## 7. Получение данных

Рассмотреть:

- Polling
- Long Polling
- Streaming

## 8. Деплой

Описать стратегии:

- Rolling Release
- Blue/Green
- Canary Release

Кратко ответить:

- какие паттерны использованы и зачем
- как обеспечена отказоустойчивость
- как масштабируется система
