# Лабораторная работа №2

## Индексы, транзакции, уровни изоляции, кэширование

### Задание

### Часть 1. Работа с индексами

1. Создать таблицу `users` с полями: `id` — уникальный идентификатор, `name` — имя пользователя, `email` — email пользователя.
2. Наполнить таблицу 10 000+ записями.
3. Выполнить поиск пользователя по email **без индекса** и зафиксировать время выполнения и количество проверенных строк.
4. Создать индекс по полю `email`.
5. Выполнить тот же поиск с индексом, сравнить время и количество проверенных строк.

**Что продемонстрировать:**

* Разницу между полным сканированием таблицы и использованием индекса.
* Как индекс влияет на производительность запросов.

### Часть 2. Транзакции и откат (ACID)

1. Создать таблицу `accounts` с полями: `id` — уникальный идентификатор, `balance` — баланс счета.
2. Создать метод/функцию перевода денег между счетами:
* Списать деньги с одного счета
* Зачислить деньги на другой
* Искусственно вызвать ошибку после списания денег


3. Выполнить перевод и зафиксировать состояние счетов **до** и **после**.

**Что продемонстрировать:**

* При ошибке нет изменений в базе -> демонстрация свойства **Atomicity** из ACID.

### Часть 3. Уровни изоляции

1. Настроить уровни изоляции (например, `READ COMMITTED`) для транзакции перевода.
2. Создать два параллельных процесса/транзакции:
* Одна изменяет баланс, но ещё не завершена
* Другая читает баланс


3. Зафиксировать поведение базы при разных уровнях изоляции.

**Что продемонстрировать:**

* Как уровень изоляции влияет на видимость промежуточных данных.

### Часть 4. Кэширование

1. Настроить внешнее кэш-хранилище (Redis или аналог).
2. Реализовать стратегию **Cache Aside**:
* При запросе пользователя сначала проверяется кэш
* Если данных нет — читаем из БД и помещаем в кэш


3. Повторный запрос по тому же ключу должен брать данные из кэша.

**Что продемонстрировать:**

* Снижение нагрузки на базу
* Эффективность кэширования
