# Лабораторная работа 2

Тема: индексы, транзакции, уровни изоляции, кеширование.

## Что внутри
- `PostgreSQL` для частей 1-3
- `Redis` для части 4
- `Python`-скрипт, который последовательно выполняет все сценарии и печатает результаты

## Структура
- `docker-compose.yml`
- `app/Dockerfile`
- `app/requirements.txt`
- `app/lab2.py`

## Запуск
```bash
docker compose up --build
```

После запуска контейнер `app` выполнит лабораторную и завершится.

## Что демонстрируется

### Часть 1. Индексы
1. Создается `users(id, name, email)`.
2. Добавляется 20 000 записей.
3. Запрос по `email` выполняется без индекса (`EXPLAIN ANALYZE`, фиксируются тип плана, время, число проверенных строк).
4. Создается индекс `idx_users_email`.
5. Тот же запрос выполняется с индексом и сравнивается результат.

### Часть 2. Транзакции и откат
1. Создается `accounts(id, balance)`.
2. Выполняется перевод между счетами внутри транзакции.
3. Искусственно выбрасывается ошибка после списания/зачисления.
4. Проверяется, что итоговые балансы не изменились (rollback, атомарность).

### Часть 3. Уровни изоляции
Показываются 2 сценария:
1. `READ COMMITTED`: повторное чтение в одной транзакции может увидеть уже закоммиченные изменения (non-repeatable read).
2. `REPEATABLE READ`: повторное чтение в той же транзакции возвращает снимок и не видит последующие коммиты.

### Часть 4. Cache Aside
1. Внешний кэш: `Redis`.
2. Функция чтения пользователя:
   - сначала проверяет кэш
   - если нет, читает из БД и кладет в кэш (`setex`)
3. Повторный запрос по тому же ключу берется из кэша.
4. В выводе видно, что количество обращений к БД уменьшается.
