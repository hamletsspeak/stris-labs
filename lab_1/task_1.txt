# Лабораторная работа 1

## Балансировка нагрузки, проксирование и кэширование

# Задание

Необходимо разработать небольшой HTTP-сервис и продемонстрировать работу
балансировки нагрузки, проксирования и кэширования.

Язык программирования и фреймворк можно выбрать **любой**.

## Часть 1. Создание сервиса

Разработать HTTP-сервис с endpoint:

GET /info

Сервис должен возвращать информацию о текущем экземпляре приложения.

Пример ответа:

{
"service": "service-1",
"time": "2026- 03 - 14T12:00:00"
}

Поле **service** должно содержать имя сервиса.

## Часть 2. Запуск нескольких экземпляров сервиса

Необходимо запустить **несколько экземпляров одного и того же сервиса**.

Минимум — **2 экземпляра**.

Пример:

```
сервис порт
service- 1 8081
service- 2 8082
```
Проверка:

[http://localhost:8081/info](http://localhost:8081/info)
[http://localhost:8082/info](http://localhost:8082/info)

Ответ должен отличаться по имени сервиса.


## Часть 3. Балансировка нагрузки

Необходимо настроить **балансировщик нагрузки** (например, Nginx). Балансировщик
должен распределять запросы между экземплярами сервиса.

Использовать алгоритм: **Round Robin**

Балансировщик должен принимать запросы по адресу:

[http://localhost:](http://localhost:)

Проверка:

[http://localhost:8080/info](http://localhost:8080/info)

При повторных запросах ответы должны приходить **от разных сервисов**.

Пример:

service- 1
service- 2
service- 1
service- 2

## Часть 4. Reverse Proxy

Балансировщик должен выполнять роль **Reverse Proxy**.

Клиент должен обращаться только к одному адресу:

[http://localhost:](http://localhost:)

Реальные сервисы (8081, 8082 и т.д.) должны быть **скрыты от клиента**.

## Часть 5. Кэширование

Добавить endpoint:

GET /data?id=

Поведение сервиса:

1. Если данные **есть в кэше** — вернуть их из кэша
2. Если данных **нет** — сгенерировать значение и сохранить его в кэш

Пример ответа:

{
"id": 1,
"value": "Random data",
"source": "generated"
}


Повторный запрос:

{
"id": 1,
"value": "Random data",
"source": "cache"
}
